# 비밀번호 보기/숨기기와 복사 버튼을 넣으며 신경 쓴 것

## 이번에 한 작업

비밀번호가 표시되는 화면들에 보기/숨기기 기능과 복사 버튼을 추가했다.

처음에는 눈 모양 아이콘 하나를 붙이면 되는 UI 작업처럼 보였다. 하지만 비밀번호 화면은 실수로 노출되면 안 되고, 값이 없는 경우나 긴 값이 들어온 경우에도 화면이 깨지면 안 된다.

내가 실제로 수정한 부분은 다음과 같았다.

- 비밀번호 보기/숨기기 아이콘 추가
- 아이콘 디자인과 위치 조정
- 마우스를 클릭하고 있는 동안만 비밀번호가 보이도록 변경
- 비밀번호가 없는 칸은 빈 문자열처럼 보이도록 처리
- 비밀번호 복사 버튼 추가
- 지정 비밀번호 등록/수정 화면에도 입력값 숨김 처리 추가

## 클릭 중에만 보이게 한 이유

처음에는 토글 방식으로 볼지, 누르고 있는 동안만 볼지 고민했다.

비밀번호 화면에서는 사용자가 확인을 마친 뒤 다시 숨기는 걸 잊을 수 있다. 그래서 버튼을 누르고 있는 동안만 보이고, 마우스를 떼면 다시 숨겨지는 방식으로 정리했다.

이 방식은 사용성이 조금 더 엄격하지만, 운영 화면에서는 노출 시간을 줄이는 장점이 있었다.

## 레이아웃 문제

긴 비밀번호가 들어오면 아이콘 위치가 밀리거나, 복사 버튼이 흔들리는 문제가 생길 수 있다.

그래서 비밀번호 표시 영역과 버튼 영역을 분리해서, 값의 길이에 따라 버튼 위치가 움직이지 않도록 조정했다. 비밀번호가 없는 경우에는 `null`이나 `undefined` 같은 값이 보이지 않도록 빈 문자열로 처리했다.

## 복사 버튼에서 본 점

복사 버튼은 편하지만, 실패할 수도 있다.

브라우저 권한, HTTPS 여부, 클립보드 API 지원 여부에 따라 동작이 달라질 수 있기 때문이다. 그래서 복사 성공/실패 메시지를 나누고, 실패해도 화면 전체가 깨지지 않게 처리하는 기준으로 봤다.

## 정리

비밀번호 보기/복사 UI는 작은 편의 기능처럼 보이지만, 실제로는 보안 노출 시간과 사용성 사이의 균형을 잡는 작업이었다.

비밀번호 화면을 만들 때는 "보이게 할 수 있는가"보다 "언제, 얼마나, 어떤 조건에서 보이게 할 것인가"를 먼저 정해야 한다.
