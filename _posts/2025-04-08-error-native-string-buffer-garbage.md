# 네이티브 연동에서 긴 문자열 뒤에 쓰레기값이 붙던 문제

## 문제

네이티브 라이브러리와 연동하는 기능에서 일정 길이 이상의 문자열을 넘기면 뒤에 의미 없는 값이 붙는 문제가 있었다.

처음에는 입력값 검증이나 화면 표시 문제처럼 보였다. 하지만 짧은 문자열은 정상이고, 특정 길이를 넘는 문자열에서만 깨지는 패턴이라 네이티브 경계의 문자열 처리 문제로 보고 추적했다.

## 내가 확인한 것

확인한 지점은 다음과 같았다.

- Java 쪽에서 넘기는 문자열은 정상인지
- JNI 경계에서 문자열 인코딩과 길이가 어떻게 처리되는지
- C/C++ 쪽 버퍼가 충분한지
- 문자열 종료 문자가 명확히 들어가는지
- 로그 출력 시 실제 값과 버퍼 잔여값이 섞이는지

문제는 비즈니스 로직보다 문자열을 옮기는 경계에 가까웠다.

## 수정 방향

수정은 문자열 길이와 종료 처리를 명확히 하는 방향으로 잡았다.

```text
입력 문자열 확인
-> 네이티브 함수로 전달
-> 버퍼 크기와 복사 길이 제한
-> 문자열 종료 보장
-> 결과 로그 확인
```

네이티브 연동에서는 문자열이 "보기에 정상"이어도 내부 버퍼에는 이전 값이 남아 있을 수 있다. 그래서 복사할 길이와 종료 위치를 명확히 처리해야 했다.

## 검증한 케이스

아래처럼 길이를 나눠 확인했다.

1. 짧은 문자열
2. 경계 길이에 가까운 문자열
3. 경계보다 긴 문자열
4. 특수문자가 포함된 문자열
5. 반복 호출 시 이전 값이 섞이지 않는지

## 정리

Java와 C/C++ 경계에서 생기는 문제는 화면이나 서비스 로직만 보면 원인을 찾기 어렵다.

문자열 길이, 인코딩, 버퍼 크기, 종료 문자를 같이 봐야 한다. 특히 보안 값이나 설정값처럼 정확한 문자열이 중요한 기능에서는 네이티브 경계 테스트를 별도로 해야 한다.
