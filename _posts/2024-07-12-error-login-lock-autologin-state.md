## 문제

로그인을 여러 번 실패해 일시적으로 비활성화된 계정에 자동 로그인을 시도하면, 계정 상태가 관리자에 의해 비활성화된 계정처럼 바뀌는 문제가 있었다.

이렇게 되면 일정 시간이 지나면 풀려야 하는 잠금 상태가 풀리지 않는다. 사용자 입장에서는 잠시 후 다시 로그인하면 되는 상황이 영구 비활성화처럼 보이게 된다.

## 내가 확인한 것

계정 비활성화에는 이유가 여러 가지 있었다.

- 로그인 실패 횟수 초과로 인한 임시 잠금
- 관리자가 직접 비활성화
- 장기 미접속 정책에 따른 비활성화
- 존재하지 않는 계정으로 자동 로그인 시도

문제는 자동 로그인 흐름에서 이 상태들을 충분히 구분하지 못한 것이다.

## 수정 방향

수정은 계정 상태값을 더 세밀하게 보고, 자동 로그인 실패가 기존 상태를 덮어쓰지 않게 하는 방향으로 진행했다.

```text
계정 조회 -> 현재 비활성화 사유 확인 -> 자동 로그인 가능 여부 판단 -> 상태 덮어쓰기 방지
```

로그인 실패로 인한 잠금은 시간이 지나면 해제될 수 있어야 한다. 반면 관리자가 비활성화한 계정은 자동으로 풀리면 안 된다. 두 상태를 같은 값처럼 다루면 안 됐다.

## 같이 수정한 것

이 흐름을 보면서 관련 안내 문구와 로그도 같이 정리했다.

- 비활성화된 계정 자동 로그인 시 안내 문구 수정
- 존재하지 않는 계정 자동 로그인 시 500 오류가 나지 않게 처리
- 로그인 실패 5회 이상 관련 로그 추가
- 연락처 입력 형식 제한 정리

## 정리

계정 상태는 단순히 활성/비활성으로만 보면 부족하다.

왜 비활성화됐는지에 따라 해제 조건과 안내 문구가 달라진다. 자동 로그인처럼 별도 경로로 들어오는 기능은 기존 로그인 실패 상태를 덮어쓰지 않도록 특히 조심해야 한다.
