# 무결성 검증 기능과 이력 화면을 추가하며 나눈 역할

## 이번에 한 작업

보안 요구사항 대응을 위해 무결성 검증 기능과 검증 이력 화면을 추가했다.

무결성 검증은 단순히 버튼을 눌러 결과를 보는 기능이 아니다. 어떤 파일이나 값이 기준과 달라졌는지 확인하고, 그 결과를 나중에 점검할 수 있도록 이력으로 남겨야 한다.

내가 실제로 손댄 범위는 다음과 같았다.

- 무결성 검증 이력 화면 추가
- 검증 결과를 저장하는 모델과 매퍼 추가
- 관리 메뉴에서 이력 화면으로 이동할 수 있게 연결
- 검증 실행 결과와 조회 결과를 분리
- 다국어 메시지와 안내 문구 정리
- 특정 환경에서는 요구사항에 맞게 기능 노출 여부를 조정

## 역할을 나눈 이유

무결성 검증 기능은 세 가지 역할로 나눠야 했다.

| 역할 | 설명 |
| --- | --- |
| 검증 실행 | 현재 파일이나 설정이 기준과 같은지 확인 |
| 결과 저장 | 누가, 언제, 어떤 결과를 확인했는지 기록 |
| 이력 조회 | 관리자가 나중에 점검할 수 있게 화면 제공 |

실행 결과만 화면에 보여주고 끝내면 감사 자료로 쓰기 어렵다. 반대로 이력만 저장하고 화면이 없으면 운영자가 점검하기 어렵다.

## 기능 노출을 조정한 이유

보안 기능은 모든 환경에서 같은 방식으로 노출되지 않을 수 있다.

요구사항이나 배포/운영 환경에 따라 자체 검사, 업데이트, 무결성 검증 같은 메뉴를 보이게 하거나 숨겨야 하는 경우가 있었다. 그래서 메뉴와 기능을 완전히 삭제하기보다, 환경에 따라 노출을 조정할 수 있는 방향으로 정리했다.

## 정리

무결성 검증은 "검증 로직"보다 "검증 결과를 어떻게 남기고 보여줄 것인가"가 중요했다.

보안 요구사항 대응 기능은 실행, 저장, 조회, 메뉴 노출 기준을 분리해 두어야 나중에 환경별 요구사항을 맞추기 쉽다.
