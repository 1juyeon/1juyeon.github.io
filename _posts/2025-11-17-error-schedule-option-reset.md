# 스케줄 활성화 토글에서 옵션 상태가 초기화되던 문제

## 문제

스케줄 항목을 비활성화했다가 다시 활성화하면, 사용자가 저장해둔 일부 옵션이 기본값으로 돌아가는 문제가 있었다.

스케줄 자체는 남아 있었기 때문에 처음에는 데이터가 정상처럼 보였다. 하지만 다시 켜는 순간 제외 조건 같은 부가 옵션이 풀리면서, 사용자가 원래 의도한 실행 조건과 다른 상태가 됐다.

이 문제는 특정 기능 하나의 문제가 아니라, **토글 UI에서 저장된 상태를 어떻게 복원할 것인가**에 가까웠다.

## 내가 확인한 것

처음에는 저장 로직 문제라고 생각했다. 옵션이 DB에 저장되지 않아서 사라지는 줄 알았다.

하지만 확인해보니 저장된 값은 남아 있었고, 비활성 상태에서 활성 상태로 바뀌는 화면 흐름에서 기존 옵션을 다시 읽지 못하거나 초기화 코드가 기본값으로 덮어쓰는 쪽에 가까웠다.

확인한 지점은 다음과 같았다.

- 스케줄 저장 시 옵션 값이 실제로 남는지
- 비활성화할 때 옵션 값까지 변경되는지
- 다시 활성화할 때 기존 옵션을 조회하는지
- 화면 초기화 스크립트가 기존 값을 기본값으로 덮어쓰는지
- 공백이 포함된 스케줄명에서 팝업이 반복되는 다른 UI 문제가 같이 있는지

## 수정 방향

수정은 "활성화하면 새 스케줄처럼 초기화"가 아니라 "기존 스케줄 상태를 복원"하는 방향으로 잡았다.

```text
기존 스케줄 조회
-> 활성 상태 변경
-> 저장된 옵션 유지
-> 화면에 기존 옵션 재반영
```

핵심은 토글 버튼의 역할을 명확히 하는 것이었다.

활성/비활성 토글은 실행 여부만 바꾸는 기능이지, 사용자가 저장한 옵션을 새로 만드는 기능이 아니다. 그래서 활성화 처리에서 기존 옵션 값을 다시 읽고, 화면 초기화 코드가 그 값을 덮어쓰지 않도록 조정했다.

## 검증한 케이스

아래 흐름으로 확인했다.

1. 제외 옵션을 켠 스케줄을 저장한다.
2. 해당 스케줄을 비활성화한다.
3. 다시 활성화한다.
4. 저장했던 제외 옵션이 유지되는지 확인한다.
5. 다른 제외 옵션도 같은 방식으로 확인한다.
6. 스케줄명에 공백이 있을 때 팝업이 반복되지 않는지 확인한다.

## 정리

이 작업을 하면서 다시 느낀 건, 토글 UI는 보기보다 상태 관리가 까다롭다는 점이다.

사용자는 비활성화가 "잠시 꺼두기"라고 생각한다. 그런데 다시 켰을 때 옵션이 초기화되면 사용자는 데이터를 잃었다고 느낀다.

그래서 스케줄이나 설정 화면을 만들 때는 저장값, 화면 초기화, 활성화 처리, 부가 옵션 복원을 따로 확인해야 한다. 토글은 상태 하나만 바꾸는 것처럼 보여도, 실제로는 기존 설정을 보존하는 계약까지 포함한다.
