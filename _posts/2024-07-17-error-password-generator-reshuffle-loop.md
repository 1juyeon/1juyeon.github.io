## 문제

비밀번호 랜덤 생성 과정에서 특정 조건 조합일 때 재섞기 로직이 계속 반복되는 문제가 있었다.

보안 조건을 강화하면서 동일 문자 반복, 키보드 순차 입력, 직전 비밀번호 재사용 같은 검사를 추가했는데, 생성된 문자열이 조건을 통과하지 못하면 다시 섞는 흐름이 있었다. 문제는 일부 조건에서 통과 가능한 문자열이 쉽게 나오지 않아 반복이 길어지는 것이었다.

## 이번에 같이 추가한 조건

비밀번호 보안성 기준도 같이 보강했다.

- 사용자 계정과 동일한 비밀번호 금지
- 같은 문자나 숫자가 연속 반복되는 패턴 금지
- 키보드에서 연속된 문자나 숫자 순서 금지
- 키보드 연속 문자열의 역순도 금지
- 직전에 사용한 비밀번호 재사용 금지

조건이 늘어나면 생성기는 더 많은 후보를 탈락시킨다. 그래서 검증 로직만 강화하면 생성 로직이 멈출 수 있다.

## 수정 방향

무한히 재섞는 방식은 위험하다고 보고, 일정 횟수를 넘으면 문자열을 새로 생성하도록 바꿨다.

```text
후보 생성 -> 조건 검증 -> 실패 시 재섞기 -> 제한 횟수 초과 시 재생성
```

이렇게 하면 특정 후보 문자열이 계속 실패하는 상태에서 빠져나올 수 있다.

## 같이 본 케이스

지정 규칙일 때 비밀번호 생성 조건 체크박스가 선택되지 않았다고 뜨는 오류도 함께 확인했다.

규칙 종류에 따라 필요한 조건이 다른데, 화면에서는 모든 경우에 같은 체크박스 기준을 요구하면 안 됐다. 그래서 지정 규칙과 랜덤 생성 규칙의 검증 기준을 나눠서 봤다.

## 정리

비밀번호 정책은 강하게 만들수록 생성 로직도 같이 어려워진다.

조건 검증만 추가하면 보안은 강화된 것처럼 보이지만, 실제 생성기가 끝나지 않거나 사용자가 이해하기 어려운 오류를 볼 수 있다. 정책 검증과 생성 알고리즘은 항상 같이 테스트해야 한다.
