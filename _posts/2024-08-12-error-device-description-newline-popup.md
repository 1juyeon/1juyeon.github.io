## 문제

외부 데이터를 선택하는 팝업에서 목록을 받아오지 못하는 문제가 있었다.

처음에는 조회 API나 권한 문제처럼 보였다. 하지만 특정 데이터가 포함될 때만 팝업이 깨졌고, 원인을 따라가 보니 설명 필드에 들어간 줄바꿈 문자가 화면 스크립트나 데이터 렌더링 흐름을 깨고 있었다.

## 내가 확인한 것

확인한 지점은 다음과 같았다.

- 장치 목록 데이터가 서버에서 정상 조회되는지
- 팝업 화면까지 데이터가 전달되는지
- 특정 장치가 포함될 때만 실패하는지
- 장치 설명 문자열에 줄바꿈이나 특수 문자가 포함되어 있는지
- 화면에서 문자열을 그대로 렌더링하면서 스크립트가 깨지는지

데이터 조회 자체는 문제가 없었고, 화면에 전달된 문자열을 처리하는 과정이 문제였다.

## 수정 방향

장치 설명에 줄바꿈 기호가 들어오면 공백으로 치환하도록 수정했다.

```text
원본 설명 -> 줄바꿈 문자 제거/공백 처리 -> 팝업 렌더링
```

중요한 건 데이터를 무조건 삭제하는 것이 아니라, 화면을 깨지 않도록 표시용 문자열을 안전하게 만드는 것이었다.

## 같이 수정한 UI 문제

예외 장치가 몇 개 등록되어 있는지와 상관없이 화면에 빈 줄이 고정으로 생기는 문제도 같이 보완했다.

팝업이나 목록 화면은 데이터가 없을 때, 1개일 때, 여러 개일 때가 모두 자연스럽게 보여야 한다. 고정 줄 수를 기준으로 만들면 실제 데이터 수와 화면이 어긋날 수 있다.

## 정리

외부에서 들어오는 설명 필드는 생각보다 위험하다.

줄바꿈 하나도 화면 렌더링을 깨거나 스크립트 문자열을 망가뜨릴 수 있다. 목록 팝업에서는 서버 조회뿐 아니라 표시용 문자열 정리와 빈 데이터 UI까지 같이 확인해야 한다.
