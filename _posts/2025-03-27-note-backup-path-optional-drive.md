# 백업 경로 설정에서 불필요한 드라이브 의존성을 줄인 기록

## 이번에 한 작업

백업 위치를 특정 드라이브로 설정했을 때, 다른 드라이브의 백업 폴더까지 반드시 지정해야 하는 문제를 수정했다.

운영자가 백업 경로를 하나만 쓰기로 선택했는데도 사용하지 않는 경로를 요구하면 설정 경험이 이상해진다. 더 큰 문제는 실제로 필요 없는 폴더가 없다는 이유로 백업 설정이나 검증이 실패할 수 있다는 점이다.

## 내가 확인한 것

백업 경로 검증 흐름을 확인했다.

- 선택한 백업 드라이브가 어디인지
- 사용하지 않는 보조 경로까지 필수로 검사하는지
- 폴더가 없을 때 생성 대상인지, 오류 대상인지
- 백업 스케줄 설정과 실제 백업 실행이 같은 경로 기준을 쓰는지
- 경로가 비어 있을 때 안내 메시지가 적절한지

문제는 선택하지 않은 경로까지 필수값처럼 다루는 데 있었다.

## 수정 방향

수정 방향은 선택된 백업 위치만 필수로 보고, 사용하지 않는 경로는 검증 대상에서 제외하는 것이었다.

```text
백업 위치 선택 -> 선택된 경로만 필수 검증 -> 실행 경로 반영
```

이렇게 하면 운영자가 특정 드라이브만 쓰는 구성에서도 불필요한 폴더를 만들 필요가 없다.

## 정리

설정 검증은 모든 입력칸을 무조건 필수로 보는 것이 아니라, 사용자가 선택한 운영 방식에 맞게 달라져야 한다.

백업 기능에서는 경로 검증, 스케줄 설정, 실제 실행 경로가 같은 기준을 써야 한다. 그래야 화면에서는 저장됐지만 실행 시 다른 경로를 보는 문제가 생기지 않는다.
