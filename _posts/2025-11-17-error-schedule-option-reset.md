# 비밀번호 변경 일정 옵션이 다시 켜질 때 풀리는 문제

## 문제

비밀번호 변경 일정을 비활성화했다가 다시 활성화하면, 주말이나 공휴일 제외 옵션이 풀리는 문제가 있었다.

일정 자체는 정상적으로 남아 있는데, 다시 켜는 순간 일부 옵션이 기본값처럼 돌아가면 운영자는 원래 의도와 다른 일정으로 작업을 실행하게 된다.

## 내가 확인한 것

처음에는 저장 로직 문제라고 생각했다. 하지만 확인해보니 저장된 값이 사라진 것이 아니라, 비활성 상태에서 다시 활성 상태로 바꾸는 UI 흐름에서 옵션 값이 제대로 복원되지 않는 쪽에 가까웠다.

같이 확인한 항목은 다음과 같았다.

- 일정 저장 시 옵션 값이 DB에 남는지
- 비활성화할 때 옵션 값까지 변경되는지
- 활성화할 때 기존 옵션을 다시 읽는지
- 화면 초기화 스크립트가 기본값으로 덮어쓰는지
- 공백이 포함된 일정명에서 팝업이 반복되는 다른 UI 문제가 있는지

## 수정 방향

수정은 "활성화하면 새 일정처럼 초기화"가 아니라 "기존 일정 상태를 복원"하는 방향으로 잡았다.

그래서 활성화 처리 시 기존 옵션 값을 다시 읽고, 화면 초기화 코드가 이 값을 덮어쓰지 않게 조정했다.

```text
기존 일정 조회 -> 활성 상태 변경 -> 기존 제외 옵션 유지 -> 화면 반영
```

## 검증한 케이스

검증은 아래 흐름으로 했다.

1. 주말 제외 옵션을 켠 일정을 저장한다.
2. 일정을 비활성화한다.
3. 다시 활성화한다.
4. 주말 제외 옵션이 유지되는지 확인한다.
5. 공휴일 제외 옵션도 같은 방식으로 확인한다.
6. 일정명에 공백이 있을 때 팝업이 반복되지 않는지 확인한다.

## 정리

일정 기능에서 중요한 건 실행 시각만이 아니다. 사용자가 선택한 제외 조건도 일정의 일부다.

비활성화와 활성화는 단순 상태 변경처럼 보여도, 기존 설정을 잃지 않는지 반드시 확인해야 한다. 특히 스케줄 화면은 저장값, 화면 초기화, 팝업 제어가 서로 영향을 주기 쉬워서 상태 복원 기준을 명확히 둬야 한다.
