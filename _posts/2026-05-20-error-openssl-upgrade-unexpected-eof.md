# OpenSSL 업그레이드 후 TLS 종료 오류가 달라진 문제

## 문제

OpenSSL 버전을 올린 뒤 일부 HTTPS 연동에서 이전과 다른 TLS 종료 오류가 나타났다.

통신 대상이 응답을 모두 보낸 뒤 연결을 닫는 방식은 환경마다 조금씩 다를 수 있다. 기존 버전에서는 넘어가던 종료 방식이 새 버전에서는 예외처럼 보이면서 연동 실패로 판단되는 케이스가 있었다.

## 내가 확인한 것

처음에는 인증서나 네트워크 문제처럼 보였다. 하지만 특정 라이브러리 버전 전환 이후에만 나타났고, 요청 자체는 대상에 도달하고 있었다.

확인한 항목은 다음과 같았다.

- TLS handshake가 성공하는지
- HTTP 응답 본문을 실제로 받았는지
- 연결 종료 시점에 어떤 오류 코드가 올라오는지
- OpenSSL 버전 차이로 종료 처리 기준이 달라졌는지
- 같은 요청을 이전 라이브러리와 새 라이브러리로 비교했을 때 차이가 있는지

## 수정 방향

수정은 "모든 EOF를 성공으로 본다"가 아니라, 예상 가능한 종료 오류를 라이브러리 옵션과 응답 검증 기준으로 분리하는 방향으로 잡았다.

```text
요청 전송
-> TLS 연결 확인
-> 응답 수신 확인
-> 종료 오류 판정
-> 실제 실패와 정상 종료 케이스 분리
```

새 OpenSSL 버전에서 예상 가능한 종료 방식을 허용하는 옵션을 검토했고, 응답을 받지 못한 실패와 응답 후 종료 과정에서 발생한 오류를 구분했다.

## 검증한 케이스

아래 케이스를 나눠 확인했다.

1. 정상 TLS 연결
2. 인증서 문제
3. 응답 수신 전 연결 종료
4. 응답 수신 후 연결 종료
5. 기존 라이브러리와 새 라이브러리의 동작 차이

## 정리

TLS 라이브러리 업그레이드는 컴파일만 통과하면 끝나는 작업이 아니다.

같은 서버와 통신해도 라이브러리 버전에 따라 오류 판정이 달라질 수 있다. 특히 보안 라이브러리 업그레이드에서는 handshake, 응답 수신, 연결 종료를 나눠서 봐야 한다.
